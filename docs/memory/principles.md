# 運作原則

**建立日期**: 2025-11-04  
**最後更新**: 2025-11-04

---

## 開發原則

1. **MVP 出發，快速迭代**
   - 每個 Run（20 分鐘）都要能在 browser 中互動操作確認結果
   - 快速反饋，不花時間研究問題

2. **對話式協作**
   - 用戶在對話中提供 input
   - Agent 負責展開成文件
   - 不需要用戶了解目錄結構

3. **記憶優先**
   - 重要資訊必須寫入 `memory/`
   - 確保下個 Agent 不會遺漏

4. **Story 自包含**
   - Story 文件包含所有開發所需資訊
   - 開發時不需要再查閱 memory/
   - 節省 token，提高效率

---

## 文件原則

1. **文件是 Agent 的記憶**
   - 不是給人看的長期文檔
   - 是給 Agent 的交接資訊

2. **分層設計**
   - `memory/` - 核心記憶（需要時查）
   - `backlog/` - 任務管理（進行中）
   - `archive/` - 已完成/棄置（很少查）
   - `context/` - 當前上下文（快速了解）

3. **討論留底**
   - 討論過程放在 `archive/discussions/`
   - 決策結論放在 `memory/decisions/`

---

## 測試原則

1. **Agent 測試優先**
   - Run 完成前必須完成所有 Agent 可測試的項目
   - User Test 時不應該出現基礎問題（型別錯誤、crash 等）

2. **User Test 驗收**
   - 驗收 = user 在 browser 中測試
   - 通過就過，不通過就修

3. **明確告知**
   - Run 完成時必須告知用戶需要 User Test 的項目
   - 說明為什麼無法自動測試
